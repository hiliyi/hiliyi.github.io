Ubuntu 18.04 Server 开机启动指定命令

2019-07-19
运维
约 343 字 预计阅读 1 分钟 次阅读

文章目录

由于 Ubuntu 18.04 Server 的 /etc 目录下面默认没有 rc.local 文件了，所以先要自己新建一个：

1

sudo touch /etc/rc.local

编辑 rc.local 文件，输入如下内容：

#!/bin/sh -e
#
# rc.local
#
# This script is executed at the end of each multiuser runlevel.
# Make sure that the script will "exit 0" on success or any other
# value on error.
#
# In order to enable or disable this script just change the execution
# bits.
#
# By default this script does nothing.
echo "Hello Test" > /usr/local/text.log
exit 0

其中 echo “Hello Test” > /usr/local/text.log 这一行就是我们要执行的命令。

改变文件的权限：

1

sudo chmod 755 /etc/rc.local

执行重启，或者以下命令：

1

sudo systemctl restart rc.local.service

最后查看 /usr/local 目录下面 是否出现 test.log 文件，有则表明 rc.local 种设定的开机启动命令已经成功。

注1：网上有很多其他博客，说什么要改 /lib/systemd/system 目录的 rc-local.service 文件，还要链接到 /etc/systemd/system 目录，但是经过我自己的测试，这些都无必要。只要自建 rc.local 文件，改好权限就够了。

注2： sudo systemctl restart rc.local.service 命令也可以改成 sudo systemctl restart rc-local.service，其实 rc.local.service 是一个指向 rc-local.service 的链接，它们都位于 /lib/systemd/system 目录下。
